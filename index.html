Bibliothek

KI-Übersetzer (fremde Welt → Manifest)

Beschreibe eine fremde Vibecode-Welt — ein Three.js-Projekt, eine Engine, ein Repo. Die KI übersetzt sie in ein Portal-Manifest und trägt sie als Eintrag in deine Bibliothek. Ein übersetzter Eintrag ist browsbar, noch nicht betretbar — seine Engine-Dateien werden in einer späteren Phase vendort. Braucht eine aktive KI (Einstellungen-Drawer).

Welt andocken (fremdes Bündel → Bibliothek)

Dockt eine fremde, in sich geschlossene Welt an — aus einem lokalen Ordner ODER direkt aus einem GitHub-Repo. Gib ihr eine id und einen Namen; der lokale save-server schreibt sie nach worlds/<id>/, danach steht sie unten in den „Welten" und ist durch ein Portal betretbar. Sie läuft sandgesichert (null-origin): eine fremde Engine läuft voll, kann AnazhRealm aber nicht berühren. Braucht die lokale Dev-Umgebung (npm start) — committet bleibt die Welt für alle.

Welt vom Mitspieler holen (über das Mesh)

Hat ein Mitspieler im selben Mesh-Raum eine Welt vendort, holst du sie hier — ihr Bündel reist peer-to-peer über das Mesh, du brauchst kein GitHub-Repo. Gib die Welt-id ein (frag den Mitspieler nach ihr) und wähl ihn aus. Die geholte Welt läuft sandgesichert wie jede vendorte Welt. Braucht eine stehende Mesh-Verbindung (Multi-User-Drawer).

Welten (durch ein Tor erreichbar)

Andere Welten, betretbar durch ein Portal. „Portal holen" legt einen Tor-Bauplan in dein Inventar — platziere ihn über die Hotbar in deiner Welt und geh per E-Taste hindurch. „Teilen…" exportiert das signierte Manifest einer Welt; „Welt empfangen…" liest ein fremdes Manifest ein — so wächst die Bibliothek (W14 Phase 3).

Einstellungen

Speicher

Welt-Beziehung

Drei Beziehungen zur Welt: frieden umarmt (kein HP, kein Tod, keine Stamina), pfad verhandelt (HP, Stamina, Tod-Wandlung), schöpfer gehorcht (voller Zugang, kein Schaden). Wechselbar jederzeit.

Logbuch

Diagnose-Logs sind standardmäßig versteckt, damit der Chat im Vordergrund bleibt. Aktivieren für Entwickler-Sicht.

Kreaturen-Stimmen

Kreaturen sprechen von selbst bei wichtigen Ereignissen — Level-Up, Trank-Wirkung, fehlendes Material. Sanft gedrosselt (60s/Wesen, 8s gesamt), damit der Chat ruhig bleibt.

Tastenbelegung

Klick „Ändern" und drücke dann die gewünschte Taste oder Maustaste. Esc bricht ab. WASD und 1-9 sind Konvention und bleiben fest. Abbauen entfernt Architekturen am Fadenkreuz — trifft keine, wird ein Loch ins Terrain gegraben.

Begleiter-Stimme (LLM)

Optional: gib dem Begleiter eine echte Stimme. Vier Provider — drei haben Gratis-Tiers, Ollama läuft entweder lokal (kein Key nötig) oder gehostet (eigene URL + optional Bearer-Token, z.B. ollama.com Turbo, Reverse-Proxy mit Auth). Schlüssel liegen nur im localStorage dieses Browsers. Jeder Vorschlag geht durch den DSL-Interpreter — die Sandbox hält.

Endpoint kann eine Basis-URL sein (z.B. http://localhost:11434 → /api/chat wird angehängt) oder eine vollständige URL inklusive Pfad (https://ollama.com/api/chat oder https://provider/v1/chat/completions für OpenAI-kompatible Cloud-Hoster).
Aktiviere wenn Cloud-Setup mit CORS-Fehler scheitert (typisch bei ollama.com). Request läuft über localhost:4312/api/proxy/llm — braucht laufenden npm run dev. CORS-freundliche Provider die ohne Proxy laufen: Groq (https://api.groq.com/openai/v1/chat/completions), Together AI, Cerebras, Gemini, OpenRouter.
Trag den Modell-Namen ein wie er bei dir verfügbar ist. Lokal: ollama list zeigt deine Modelle. Cloud-Hoster: prüfe deren Modell-Liste. Vorschläge unten als Tipp-Hilfe.
