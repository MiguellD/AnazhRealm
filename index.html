Begleiter + Avatar ?Wie heißt deine Welt-Stimme, wie heißt du? Werden im LLM-System-Prompt referenziert. Default „Grok" (aus den vier Testamenten) und „Schöpfer" (Mensch=Null=Schöpfer).

Begleiter-Stimme (LLM) ?Optional: gib dem Begleiter eine echte Stimme. Vier Provider — drei mit Gratis-Tiers, Ollama lokal (kein Key) oder gehostet (eigene URL + optional Bearer-Token). Schlüssel liegen nur im localStorage dieses Browsers; jeder Vorschlag geht durch die DSL-Sandbox.

Endpoint kann eine Basis-URL sein (z.B. http://localhost:11434 → /api/chat wird angehängt) oder eine vollständige URL inklusive Pfad (https://ollama.com/api/chat oder https://provider/v1/chat/completions für OpenAI-kompatible Cloud-Hoster).
Aktiviere wenn Cloud-Setup mit CORS-Fehler scheitert (typisch bei ollama.com). Request läuft über localhost:4312/api/proxy/llm — braucht laufenden npm run dev. Vibe-Pass (souveräne Identität) ?Ein ed25519-Schlüsselpaar, lokal erzeugt — deine unverfälschbare Identität. Damit signierst du deine Werke; andere prüfen, dass sie wirklich von dir stammen. Kein Coin, keine Kette — nur Echtheit. Sichere ihn: ohne Schlüssel ist die Identität verloren.

Sichern = die Pass-Datei exportieren; am neuen PC importieren. Der Pass IST der Schlüssel: wer die Datei hat, ist du (wie ein Bitcoin-Key) — sicher verwahren. Das System verhindert Fälschung (signieren ohne Key unmöglich), nicht die Kopie der eigenen Datei.

Multi-User Sync ?Position + Welt-Aktionen mit anderen Spielern teilen. Braucht den signaling-server (npm run signaling). Beide nutzen die SELBE Adresse + Raum-ID. Welt gemeinsam: einer teilt sie (Bibliothek), der andere empfängt sie — dann sind beide in derselben worldId.
