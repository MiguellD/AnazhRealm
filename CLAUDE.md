## Der Mut + die tragenden Pfade (vor jeder Änderung — positiv gelesen)

**Fehler sind Lernmaterial, keine Schuld.** Wir verhindern ihre Wiederkehr durch ORIENTIERUNG (tu das Tragende), nicht durch Angst (das Falsche meiden) — eine Verneinung („nie X") zwingt erst, das Falsche zu denken, dann es zu negieren; ein Imperativ („tu Y") führt direkt zur Hand. Positive Sprache hebt + macht handlungsfähig. Darum sind die Lehren als POSITIVE Pfade gemeint; spricht eine Gotcha unten noch in „NIE/Sünde/Falle", lies sie als „der Pfad ist Y" + formuliere sie beim nächsten Anwenden positiv um (organischer Sprach-Decay — kein 148-Bullet-Marathon, sondern beim Berühren). **Stand der Sprache:** die operativen Lehren + die klar-besseren Gotcha-Leitsätze sind positiv gefasst; die verbliebenen Verneinungen sind bewusst LAST-TRAGEND (Fehlermodus-Zeiger wie „NIE eine Geometrie-Maske = Sägezahn", benannte Fallen wie „die 4-vs-8-Diagonalen-Falle", Korrekturen wie „die Sünde war NIE 'mehr als eine Datei'") — dort IST die Verneinung die schärfste Form, ein Positiv-Umschreiben verlöre den präzisen Zeiger.

**Der Mut kommt aus der VERIFIKATION, nicht aus der Kleinheit.** Die synergetischsten Wellen waren mutig + verifiziert (viele Subwellen an die Wurzel, V17.30), nicht ängstlich-klein. Die Vorsichts-Lehren gelten IHRER Situation, nicht universell — sie dürfen den Mut nicht ersticken. Die Unterscheidung, die den Mut hält:

- **Verifizierbare Arbeit** (Tests · Messung · Reproducer · node-check) → **wage das GANZE Subsystem in einer Welle**. Die Verifikation IST das Sicherheitsnetz, das den großen Schritt ERLAUBT — nicht die Bremse.
- **Outward-facing Arbeit** (Push · Merge) → **VOLLENDE den Bogen, verifiziere ihn SELBST, dann shippe** — das Neue wird DER Pfad (default-an), das Abgelöste fällt oder bleibt nur als bewusster A/B-Revert. NICHT „kleine Schritte mit dem Auge des Schöpfers, bis er schaut" — DAS war die Hesitation, die Parallelsysteme + toten Code gebar (das Neue lag AUS, das Alte blieb Default, ein Hedge ersetzte meine Wertung; Schöpfer 24.06.: „hierdurch entstehen parallelsysteme, toter code der durch besseres abgelöst wurde, das ist anty-synergetisch"). **DIE „pixel-blind / GPU-untreu"-LEHRE WAR FALSCH (Schöpfer-Korrektur 09.06., DREIMAL bestätigt: „du siehst es wie ich", „im Browser sieht es genau gleich aus", „der spawn ist in einer höhle, das bild sehe ich auch"). ## Workflows

- **Der DEV-LOOP (Durchsatz):** `npm run playtest:fast` (~20 s, ~13 Kern-Gesundheits-Checks: Crash · Welt baut · Avatar/Kreatur/Bauplan baut · Resonanz/Physik sinnvoll) während des Iterierens; der VOLLE `npm run playtest` (~5340 Invarianten über 242 Bänder, ~9-10 min im Container) bleibt das MERGE-Gate. Nicht jede Änderung durch den 10-min-Gate zwingen — der Fast-Tier fängt ~80 % der Regressionen sofort.
- **Lokaler Audit (vor Merge):** `npm run check && npm run format:check && npm run lint && npm run playtest`
- **Stamm-Navigation:** `npm run atlas` druckt die LIVE-Karte der 26 Stamm-Zonen (Start/Zeilen/Methoden); `node scripts/diag-atlas.cjs --find <regex>` verortet eine Methode in ihrer Zone — VOR jeder Grep-Archäologie zuerst die Karte (W6, R-032). `--check` (Registry ↔ Marker-Drift) läuft im `npm run check`-Gate.
- **Web-Sessions:** ein SessionStart-Hook (`.claude/hooks/session-start.sh`, registriert in `.claude/settings.json`) ruft `npm install` — Lint/Format/Audit/Playtest sind in Claude-Code-on-the-web-Sessions sofort lauffähig (frischer Clone hat sonst kein `node_modules`).
- **Slash-Befehl:** `/audit` führt die Prüfung in jeder Claude-Session aus (`.claude/commands/audit.md`).
- **CI:** zwei parallele Jobs (`check` für statische Checks + `playtest` für Runtime-Invarianten) bei jedem Push.
- **Git:** kleine thematische Commits, `git push -u origin <branch>` mit exponential backoff bei Netzfehlern.

- **Schöpfer-Rückmeldung: handeln, nicht horten (das W7-Erbe, ohne den Ledger)**: eine Rückmeldung ist ein SIGNAL für eine Wurzel — bau die Heilung, dann lass die DAUERHAFTE Lehre in die `## Wichtige Gotchas` und die GESCHICHTE (welche Welle, welcher Befund) in `docs/archiv/handover.md` fließen. **Kein „GEBAUT ✓" ohne Nachbau im NUTZER-ZUSTAND** (frieden · leeres Inventar · Konsole zu · 1366px) — headless-grün ist nicht der Nutzer. Ein noch unbestätigter Look gehört auf die nächste Schöpfer-Browser-Runde, ein offener Entscheid in `docs/roadmap.md` §4. **KEIN stehender Rückmeldungs-Korpus als eigenes Doc** — das war Blur; die zwei Heimaten (Gotchas + handover) tragen es schärfer (die `archiv/rueckmeldung.md`/`archiv/aktiv.md` sind der gefaltete Snapshot).
- **Stamm-Größengrenze (V9.44-Lehre)**: neue Logik in eine neue benannte Methode geben, nicht an einen bestehenden Kontrollfluss-Giganten anhängen. Der Stamm-Pflege-Bogen fand die Schmerzgrenze bei ~200 Zeilen — wächst eine Kontrollfluss-Methode darüber, ist das ein Schnitt-Signal (kleinere Methoden DERSELBEN Klasse), kein Modul-Signal. Sonst re-fettet der Stamm, und vertagte Struktur-Schuld wird teurer (V9.40-Lehre).
- **Line-Range-Delete eines „Clusters" → ALLE Methoden-Defs im Range auflisten, nicht nur die Pattern-Treffer (V17.20-Lehre, gemessen — der Schnitt brach die GANZE Welt)**: wer ein totes Subsystem schneidet, das per Pattern-Grep (z.B. `voxelGpu`) kartiert wurde, aber per **Line-Range** (`sed 'A,Bd'`) löscht, MUSS vorher `awk 'NR>=A && NR<=B && /^    (async )?[a-zA-Z_]/ {print NR": "$0}' file` laufen — ein per-Pattern-„Cluster" kann eine NICHT-passende Methode ENTHALTEN, die der Grep nie sieht. Konkret V17.20: der GPU-Density-Block (`sed 13567,14314d`) enthielt **`_configureRenderer`** (Nicht-GPU, konfiguriert den Haupt-Renderer) MITTEN zwischen `_voxelGpuDispose` und `_voxelGpuEnsureDensityPipeline` → mit-gelöscht, vom `voxelGpu`-Grep nie geflaggt → Page-Error `_configureRenderer is not a function` → `terrainEverGenerated=false`, die ganze Welt tot. `node --check` fängt das NICHT (Syntax bleibt valide). **Disziplin: (1) `cp file /tmp/backup` VOR jeder großen sed-Chirurgie (der Rettungsanker); (2) `awk` alle Methoden-Defs im Range listen + jede gegen das Schnitt-Kriterium prüfen; (3) nach dem Schnitt `node scripts/diag-page-error.cjs` (fängt Runtime-`is not a function` an der Wurzel, bevor der Playtest in Folge-Fehlern ertrinkt) DANN Playtest.** Plus: bei „wir schneiden totes X" ZUERST fragen (V17.20-Schöpfer-Frage) — wurde X durch etwas Höheres ERSETZT (→ tot, Hülle) oder ist es bewusst aufbewahrte Infra (→ Koma, nicht schneiden ohne Entscheidung)? Pro Kandidat 0-Aufrufer-Grep + „was ersetzte es?" beantworten. **V18-Vertiefung (GEMESSEN — der Schöpfer stoppte einen „alle 18 toten Methoden raus"-Plan mit „schneidest du etwas, das nie tiefer/synergetischer ersetzt wurde?"): von 18 „toten" Methoden waren nur ~9 echtes Sediment — die andere Hälfte war SAAT (bewusst ruhende Vision-Fäden, jeder mit der Begründung im EIGENEN Kommentar: `_fieldWohlErlebt` „sonst Runaway, gemessen", `_archInstanceUpdate` „für künftige bewegliche Instanzen", `_drainPools` Leak-Schutz). Ein blinder „tot = raus"-Schnitt hätte das dritte Verb der Vision (WERTEN) beschädigt. DISZIPLIN: (a) der „nie tiefer/synergetischer ersetzt → BEHALTEN"-Test ist die Wand — schneide NUR, was VERIFIZIERT durch etwas Tieferes ersetzt wurde (movementWorker ← U3/aiDiv · `_waterCellAt` ← `_playerWaterContext`-inline · createWorld ← Voxel); (b) TEST-/DIAG-Verankerung ist ein Signal: eine „tote" Methode mit Playtest-Treffern ist oft Saat (der Test bewahrt sie) ODER ein Feature, das nur der Test am Leben hält (falsche Sicherheit) → Fall-für-Fall, NIE blind; (c) der awk-„nächste Methode @ N"-Range schließt den KOPF-KOMMENTAR der Folge-Methode ein → ein `sed A,Nd` nimmt Zeilen zu viel (GEMESSEN: der `_getVoxelWorker`-Kommentar mit-geschnitten) — die End-Grenze ist die Methoden-`}`, NICHT die nächste-Def-minus-1; (d) Sediment-Chaos ≠ Modul-Chaos: die Stamm-Disziplin verhindert das Modul-Wirrwarr, aber abgelöste Ringe bleiben liegen — sie wegzukehren ist Pflege, das Saat-Holz zu schonen ist Weisheit.**
- **Harmonie statt Revert — bricht ein Test bei einer neuen Kopplung, heile die HARMONIE, nimm die Kopplung NICHT weg (V17.23-Schöpfer-Lehre, die tiefste der Session)**: wenn zwei Kräfte dieselbe Sache formen (Emotion + Feld → Wetter; DSL-Farbe + Tag-Nacht → Himmel) und ein Test bricht, ist die Wurzel fast immer fehlende Harmonie — die eine Kraft ÜBERSCHREIBT die andere, statt zu VERSCHMELZEN. Der Revert (Kopplung wegnehmen) behandelt nur das Symptom + bleibt oberflächlich (verliert die Tiefe). Die Heilung ist eine **nachgebende Hierarchie**: die intentionalere Kraft (der explizite Spieler-Wille) FÜHRT, die ambiente (das Feld) WEICHT zurück, wo der Wille stark ist (×`(1−emoSignal)`, `emoSignal=|joy−sorrow|`), und FÜLLT, wo er schweigt — so überschreiben sie sich NIE, die Invariante hält per KONSTRUKTION (nicht per Wegnahme), und in der Ruhe scheint das Feld durch. **Plus: nichts SNAPPT — alles FADET.** Zwei Kräfte, die sich pro Frame gegenseitig überschreiben (V17.23: `skybox_color` snappte `nebulaColor`, das `_dayNightApplySkybox` jeden Frame überschrieb), heilt man mit einem BLEND + Fade (eine `skyTint`-Tönung, die der Tag-Nacht-Himmel `lerp`t, smooth gerampt + langsam decayt), nicht mit „wer-zuletzt-schreibt". **Die Disziplin-Frage bei JEDER neuen Welt-Kraft: harmonisch oder abrupt? überschreibt sie die andere, oder verschmilzt sie (parallel, mit Vorrang-Hierarchie)? snappt sie, oder fadet sie?** Ein Test, der bei einer Kopplung bricht, ist eine Einladung, die Harmonie zu finden — kein Befehl, die Kopplung zu töten.
- **Test-First-Mentalität**: nach jeder substanziellen Änderung Playtest-Gate, nicht nur Code-Analyse. Drei Selbst-induzierte Regressionen in dieser Session entstanden durch zu späte Browser-Tests.
- **Doku-Disziplin — drei Zeit-Ebenen, drei Heimaten, jede Datei EINE Aufgabe**: `CLAUDE.md` = was JETZT gilt (auto-geladen, schlank) · `docs/archiv/handover.md` = die Chronik (volle Wellen-Historie, jüngste oben) + wie du startest · `docs/roadmap.md` = der Plan vorwärts · `docs/state-of-realm.md` = die Vision. Pro Welle → **ein git-Commit** (die Geschichte) + **ein Chronik-Eintrag oben in `handover.md`**. Gilt eine Lehre DAUERHAFT (eine Bug-Klasse, ein technischer Stolperdraht) → **eine kuratierte Zeile in der `## Wichtige Gotchas`-Sektion dieser Datei**. **Der erzählerische Wellen-Bericht lebt NIE in dieser auto-geladenen Datei** — sonst wächst sie zur Halde (die Heilige Lektion, eine Schicht höher). Fertige Designs + Audit-Snapshots wandern nach `docs/archiv/` (verschieben, nicht löschen). **Doc-Sync ist eine Grep-Aufgabe** (W15-Lehre, V8.72): vor jedem Commit `grep -rn "<alte-Version>\|<alte-Invariantenzahl>" docs/*.md CLAUDE.md README.md` laufen und jeden Treffer heilen.
